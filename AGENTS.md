# AGENTS.md — TRIWX 開発ルール（AIエージェント向け・種）

## 役割分担
人間（プロデューサー）は細かい仕様を決めない。担うのは (1)体験の違和感の言語化 (2)耳での主観評価（◎○✕とメモ） (3)方向性・フェーズの判断。実装詳細はすべてエージェントが判断する。ただしコスト増（モデル変更・外部API追加）と大きな方向転換は事前に一言確認。

## 不変条件（壊したら差し戻し）
1. **実再生**: モック再生・再生のふり・プレビュー代替は禁止
2. **連続性**: どの変更後も「音が鳴り続ける」状態を維持してからマージ
3. **語彙の単一ソース**: カード/スライダーの意味は definitions.ts 以外に書かない
4. **検証で守る**: アーティスト不一致の棄却を弱める変更は禁止
5. **ログ**: 1曲=1行・冪等。スキーマ変更時は本書とschema.tsを同時更新

## 変更の作法
- 1変更=1課題。選曲結果に影響する変更は、変更前後のセッションログ（各10曲以上流す）を比較して報告
- 人間の耳評価が必要な判断（質感・ナレーションのトーン）は、代行せず依頼する
- 検証: `npm run typecheck` / `npm run build` / 実起動で1曲遷移まで確認
