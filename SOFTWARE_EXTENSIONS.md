# SOFTWARE_EXTENSIONS.md — ラジオ的情報コーナー 検討資料

> 将来構想。天気予報・交通情報・生活情報などを、音楽の流れを壊さずに曲間へ挟むための設計検討。
> コードはまだ書かない。着手時は本書を読んでから tasks/ に起票すること。
> 最終更新: 2026-07-05

## ビジョン

深夜、曲が終わると柔らかいコーナージングルが鳴り、
「TRIWXお天気です。東京地方、今夜は曇り、明け方の気温は14度」——15秒で音楽に戻る。
FMラジオが持っていた「世界とゆるくつながっている感じ」を、音楽の邪魔をしない分量で。

## 絶対原則

1. **AIに想像で作らせない。** 天気・交通など正確性が必要な情報は、外部API・公式情報の取得値のみから作る。取得値にない数値・地名・現象を一切足さない
2. **取得できなければ読まない。** APIの失敗・タイムアウト・データの鮮度切れのときは、そのコーナーを黙ってスキップし、通常のジングルや曲紹介に戻る。エラーを読み上げない、無音にもしない
3. **音楽の流れを壊さない。** 挿入は曲間のみ（曲を切らない・被せない）。1コーナーは読み上げ20秒以内。同じ曲間に情報コーナーは1つまで。コーナー同士は最低N曲空ける
4. **正時・定刻を主張しない。** 時報と同じ作法（現行の曖昧時報が先例）

## 土台は既にある

現行の**インサート機構**（`src/engine/inserts.ts`）がそのまま拡張口になる:
- `Insert.type` を増やす（`"weather" | "traffic" | "almanac"` …）
- `computeDueInserts` に発火条件を足す
- 表示テキストと読み上げテキストの分離、読み上げ設定から独立した演出音声（`speakInsert`、ゲイン増幅済み）、コーナー前の合成ジングル（`sounds.ts`）——すべて局名コール・時報で実装済みのパターンの再利用

新規に必要な土台は **「コーナーデータの取得・キャッシュ層」**（サーバ側。取得値+取得時刻を保持し、鮮度切れを判定する）のみ。

## スケジューラ設計（共通ルール案）

- 優先度: 時報 > 天気 > 交通 > 生活情報 > ジングル（同じ曲間では最優先の1つだけ）
- 頻度上限: コーナーごとに設定（例: 天気は1セッション2回まで/前回から90分）。曲間ごとの抽選ではなく「条件が揃った最初の曲間」方式（時報と同じ）
- 鮮度規則: 取得時刻から一定時間（天気60分・交通10分など）を過ぎたデータは読まない → 原則2によりスキップ
- 将来のUI: 「時報・ジングル」トグルを「ラジオ演出」設定に発展させ、コーナーごとのオンオフ

## コーナー別の検討

### 1. 天気予報（最初に作るべき本命）
- **情報源**: 気象庁の公開予報JSON（www.jma.go.jp配下、無料・公式・キー不要）を第一候補。代替: Open-Meteo（キー不要）。**着手時にエンドポイント仕様と利用条件を必ず確認**（本書の記述時点の知識は古い可能性がある）
- 地域は設定値で固定（局長の地域。位置情報の取得はしない）
- **原稿生成は2段構え**: まずテンプレート整形（取得値の埋め込みのみ、LLM不使用）で確実に。自然さが欲しくなったらLLMに「与えたフィールドの値だけを使い、数値・地名・現象を変えない」制約で言い換えさせる第2段。**テンプレ版を必ず残す**（LLM失敗時のフォールバック）
- 発火案: セッション開始後の最初の曲間+時間帯の変わり目（朝・夕）。夜間は明日の予報

### 2. 交通情報（地域依存が強く、後回し）
- **情報源候補**: 鉄道は公共交通オープンデータセンター(ODPT・要登録)や各社公式の運行情報、道路はJARTIC。**整備コストと利用規約の確認が重く、対象は局長が使う路線数本に絞るのが現実的**
- 平常運転時は読まない（「◯◯線は平常運転です」の連呼はラジオでも嫌われる）。遅延・運転見合わせがあるときだけ発火する設計にすると、原則3と情報価値が両立する
- 人間の判断事項: どの路線を対象にするか、そもそも要るか

### 3. 時報・ジングル・ステーションID（実装済み）
- 現行実装が先例。改善はROADMAP中期のT3で

### 4. 短い生活情報（正確性の担保方法で選別する）
- **計算で正確にできるもの（外部API不要・推奨）**: 日の出・日の入り時刻（緯度経度から計算可能）、二十四節気・月齢（暦計算）。「まもなく日の入り、17時32分です」は嘘のつきようがない情報で、夕暮れカードとの相性も良い
- **出典取得が必要なもの**: 「今日は何の日」（要データソース。Wikipedia等から取得できた日のみ読む）
- **明示的なフィクション**（占い等）: 正確性の原則の対象外だが、「事実と混ざる」リスクがあるため導入は局長の判断事項。入れるなら「TRIWX占い」のように虚構であることが様式で伝わる形に限る

## 失敗時の振る舞い（全コーナー共通・実装必須）

```
コーナー発火条件成立
→ キャッシュ確認（鮮度内?）
  → なし/古い → サーバへ取得（タイムアウト3秒）
    → 失敗 → 【何も読まず】通常の曲紹介へ（発火済みフラグは立てない=次の曲間で再挑戦可）
→ 成立 → コーナージングル → 読み上げ(≤20秒) → 曲紹介 → 曲
```

## 役割分担

- **人間が判断**: どのコーナーを入れるか、地域・路線の設定、頻度の体感調整、占い等フィクションの採否、各情報源の利用規約の最終確認
- **AIが実装**: 取得・キャッシュ層、テンプレ原稿、インサート拡張、コーナージングルの作曲、鮮度・頻度ルール
