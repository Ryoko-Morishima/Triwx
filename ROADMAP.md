# ROADMAP.md — TRIWX 開発ロードマップ

> 優先度順。各項目に「人間が判断すること / AIが実装すること / 完了条件」を明記。
> タスク単位の詳細は tasks/ を正とする。順序変更は人間の承認事項。
> 最終更新: 2026-07-05

## 短期（安定化・観測）

### 1. fillフォールバックの安定化と観測 → tasks/T1
- AI: OPENAI_API_KEYのtrim防御、fallbackReason/judgeRejected/sourceの発動率をログから集計できる形に
- 人間: 集計結果を見て「非常口が開きすぎていないか」の体感照合
- 完了条件: キー改行事故が構造的に再発しない。1セッション回してseed/replay発動率とゲート棄却数が数字で読める

### 2. 散らし具合レポート → tasks/T4
- AI: /api/log にセッション/期間の多様性指標（アーティスト重複率・年代分布・言語分布・source内訳）を実装
- 人間: 指標が体感と合っているかの確認、目標値を置くかの判断
- 完了条件: ブラウザで開いて散らし具合が読める。設計原則4の「客観的に測る」が実体を持つ

### 3. UIの磨き込み → tasks/T2
- AI: モバイル実機での崩れ修正、操作性の改善案の実装
- 人間: スマホで使っての違和感の列挙（これが仕様になる）
- 完了条件: 局長が日常利用して操作面の不満が出ない

### 4. ログ確認の習慣化 → tasks/T6（運用チェックリスト）
- 人間: 週1回程度 /api/log を眺める（またはAIに貼って分析させる）
- 完了条件: チェックリスト文書が存在し、局長が一人で確認を回せる

## 中期（演出・評価サイクル）

### 5. ジングル・時報の質の向上 → tasks/T3
- 人間: 現行の音・声・頻度・文言への感想（趣味の判断はすべて人間）
- AI: 音色バリエーション、音量バランス、文言調整の実装
- 完了条件: 局長が「耳に残る」と言う

### 6. 評価ログを使った改善サイクルの定型化
- AI: ✕（カード不適合）の付いた曲とカードの組を集計し、カード定義・審査プロンプトの改善案を提示する手順の確立
- 人間: ✕を付け続けること、改善案の採否
- 完了条件: 「✕が溜まる→定義が直る→✕が減る」が1周以上回った実績

### 7. /about の文面調整 → tasks/T5
- 人間: 自分の言葉への書き直し（局長の声で書かれているべき文書）
- AI: 反映とスタイル調整
- 完了条件: 局長が友人に見せて恥ずかしくないと言う

## 長期（共有・品質）

### 8. 少人数共有の体験改善
- 人間: 誰に使ってもらうか、フィードバックの収集
- AI: ユーザー別のログビュー、必要ならオンボーディング改善
- 完了条件: 局長以外の1人が自力でログイン→再生→評価まで到達
- 注意: Spotify規約で最大5人。これを超える公開は現状不可能（設計の前提）

### 9. 音声品質の向上
- 人間: 声の好みの判断（TTSモデル・ボイスの聴き比べ）
- AI: 選択肢の用意、ナレーション原稿の話し言葉最適化
- 完了条件: 局長が読み上げ常時オンで聴きたくなる

### 10. 選曲検証の高度化
- AI: 言語判定の多言語化、年データ品質の補正、審査精度の検証（LLM以外の裏取りが可能になったら検討）
- 人間: どこまでやるかの費用対効果判断
- 完了条件: 都度定義

### 11. ラジオ的情報コーナー → SOFTWARE_EXTENSIONS.md（検討資料）
- 段階: S1 天気予報（公式API・テンプレ整形・失敗時は黙ってスキップ）→ S2 計算系生活情報（日の出入り・二十四節気）→ S3 交通情報（対象路線を絞る・異常時のみ読む）
- 人間: 情報源の利用規約と地域設定の確定、頻度の体感調整、フィクション系コーナーの採否
- AI: 取得・キャッシュ層、鮮度・頻度スケジューラ、インサート拡張、コーナージングル
- 完了条件（S1）: 天気が実データのみで読まれ、API障害時に何事もなく音楽へ戻ることを実演できる
- 不変原則: AIに想像で作らせない / 取得できなければ読まない / 曲間のみ・20秒以内 / 無音にしない

### 12. ハードウェア化 → HARDWARE.md（検討資料）
- 段階: H0 キオスク（Pi+画面+スピーカー、Widevine検証が生死を決める）→ H1 物理操作（ノブ・ボタン+コントロールAPI）→ H2 物理表示（LED・針メーター・実信号VU）→ H3 木の筐体
- 先行して切り出せるソフト前提作業（中期に置ける）: カードセットのプリセット保存 / コントロールAPI / 状態のプッシュ配信
- 人間: 予算・部品購入・組み立て、librespot採否（規約判断）、音の最終評価
- AI: プリセット・コントロールAPI・GPIOデーモン・起動スクリプト・配線図と手順書
- 完了条件（H0）: 画面とスピーカーだけの置き型TRIWXが、電源投入から無操作で放送開始まで到達する
- 不変原則: 物理の針・ランプ・ノブ・メーターは本物の状態・操作のみに対応させる（飾り部品は載せない）

## やらないこと（原則により否決済み）
- 好み学習・パーソナライズ（原則1・3）
- 永久ブラックリスト（原則5）
- 機能を伴わない飾りの計器（原則7）
- モック再生・プレビュー化（原則8）
